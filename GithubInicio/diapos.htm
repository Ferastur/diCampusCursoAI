Introducción

Domina GitHub como un Pro

La herramienta esencial que separa a los aficionados de los ingenieros de software profesionales. Aprende a controlar el tiempo de tu código.

Conceptos

Git vs GitHub

Mucha gente los confunde, pero son piezas distintas de un mismo rompecabezas:

  • Git: Software local. Es el cronómetro que guarda versiones en tu disco duro.
  • GitHub: Plataforma web. El lugar donde subes esas versiones para colaborar.

Primeros Pasos

Instalar GitHub Desktop

La forma más inteligente de empezar sin memorizar comandos complejos de consola.

  • Descarga en desktop.github.com
  • Inicia sesión con tu cuenta.
  • Configura tu nombre y correo (aparecerá en cada cambio que hagas).

Estructura

¿Qué es un Repositorio?

Es la "carpeta inteligente" de tu proyecto. No solo guarda archivos, guarda su historia.

  • README.md: El manual de instrucciones.
  • .gitignore: Filtro de archivos que no deben subirse.
  • License: Quién y cómo pueden usar tu código.

El Ciclo Diario

Add, Commit & Push

Este es el mantra que repetirás cada vez que escribas código:

  1. Modify: Cambias archivos.
  2. Commit: Guardas una "foto" con un mensaje descriptivo.
  3. Push: Envías tus fotos a la nube de GitHub.

Avanzado

Ramas (Branches)

¿Quieres probar una idea loca sin romper lo que ya funciona? Usa una rama.

  • Main: La rama principal (código estable).
  • Feature branch: Tu laboratorio de experimentos.
gitGraph commit branch feature-idea checkout feature-idea commit commit checkout main merge feature-idea

Colaboración

Pull Requests

Es el corazón de la colaboración. Pides permiso para unir tu rama a la principal.

  • Tu equipo revisa el código.
  • Se discuten mejoras.
  • Se acepta (Merge) y el código se une oficialmente.
REVIEW & MERGE

Retos

Conflictos de Merge

Ocurre cuando dos personas editan la misma línea al mismo tiempo. Git no sabe cuál elegir.

GitHub Desktop te mostrará ambas versiones para que tú decidas con un clic cuál es la correcta.

Consejos

Hacerlo como un Pro

  • Commits pequeños y frecuentes (no esperes al final del día).
  • Mensajes claros: "Añado login" es mejor que "Cambios".
  • Haz Pull siempre antes de empezar para tener lo último.
git commit -m "Fix: corregido error en login"
git push origin main
