Parte 2

Tu Código al Mundo

Ya sabes usar Git. Ahora aprenderás a publicar tus proyectos, gestionar la visibilidad y convertir tus carpetas en escaparates para reclutadores y colaboradores.

Visibilidad

Público vs Privado

Elegir bien define quién puede ver y clonar tu trabajo:

  • Público: Visible para todos. Ideal para portfolios y código abierto (Open Source).
  • Privado: Solo tú y tus colaboradores invitados pueden verlo. Ideal para proyectos comerciales o tareas escolares.

En la Web

Crear Repo en GitHub.com

El proceso es sencillo pero tiene opciones clave:

  1. Click en el icono + (arriba a la derecha) -> New repository.
  2. Dale un nombre único y descriptivo.
  3. Selecciona la opción "Public".

En la Web

Inicialización Crítica

Al crear un repo público, GitHub te ofrece 3 archivos mágicos:

  • README: Crea una descripción visual inmediata.
  • .gitignore: Elige la plantilla según tu lenguaje (Python, Node, etc).
  • License: ¡Fundamental! Sin licencia, nadie puede usar legalmente tu código.

GitHub Desktop

Publicar desde la App

Si creaste el repo en tu PC primero:

  1. Abre GitHub Desktop.
  2. Haz clic en el botón azul superior "Publish repository".
  3. Desmarca la casilla "Keep this code private" si quieres que sea público.

Peligro

Secretos en Público

¡ALERTA!: Nunca, bajo ninguna circunstancia, subas contraseñas, tokens de API o claves de base de datos a un repositorio público.

Existen bots escaneando GitHub 24/7 para robar estas claves en segundos. Usa siempre archivos .env y regístralos en tu .gitignore.

Interacción

Forks, Stars y Watch

¿Cómo interactúa la gente con tu repo público?

  • Stars: Como un "Like". Guarda tu proyecto en su lista de favoritos.
  • Watch: Notifica al usuario de cada cambio o discusión.
  • Fork: Crea una copia exacta de tu repo en la cuenta del otro usuario para que pueda editarlo libremente.

Ajustes

De Privado a Público

¿Te arrepentiste? Puedes cambiarlo en cualquier momento:

  1. Ve a la pestaña Settings de tu repo.
  2. Baja hasta el final: "Danger Zone".
  3. Click en "Change visibility".

*Nota: Si pasas de público a privado, perderás las "Stars".

Comunidad

Issues y Discussions

Un repositorio público es un foro abierto:

  • Issues: Cualquiera puede reportar un error o sugerir una mejora.
  • Pull Requests externos: Extraños pueden enviarte mejoras de código que tú decides si aceptar o no.

Final

Tu Portfolio es GitHub

Los reclutadores no miran tu CV, miran tus repositorios públicos. Mantener proyectos limpios, bien documentados y públicos es la mejor forma de demostrar que sabes programar.

¡Sal ahí fuera y comparte tu código!
